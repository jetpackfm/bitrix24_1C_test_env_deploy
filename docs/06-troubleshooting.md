# 6. Устранение неполадок

## Задача `common` завершается с ошибкой

**Симптомы:** При создании пула задача `common` падает с ошибкой. В логе (`/opt/webdir/temp/common_*/status`) видны проблемы с репозиториями.

### Репозиторий Percona (prel-release)

**Ошибка:**
```text
Failed to download metadata for repo 'prel-release-noarch': Cannot download repomd.xml
```

**Причина:** В системе присутствует репозиторий `percona-prel-release`, ссылающийся на несуществующий URL.

**Решение:**
1. Найти проблемный репозиторий:
   ```bash
   ls -la /etc/yum.repos.d/ | grep prel
   ```
2. Отключить его:

   ```bash
   sed -i 's/enabled=1/enabled=0/' /etc/yum.repos.d/percona-prel-release.repo
   ```
3. Очистить кеш:

   ```bash
   dnf clean all && dnf makecache
   ```

### Репозиторий EPEL (таймаут)

Ошибка:

```text
Curl error (28): Timeout was reached for https://mirrors.fedoraproject.org/metalink?repo=epel-9...
```
**Решение:**
Заменить metalink на конкретное зеркало в файле `/etc/yum.repos.d/epel.repo`

```ini
[epel]
name=Extra Packages for Enterprise Linux $releasever - $basearch
baseurl=http://mirror.yandex.ru/epel/$releasever/Everything/$basearch/
enabled=1
gpgcheck=1
```
После внесения изменений:

```bash
dnf clean all && dnf makecache
```
### Обновление MySQL падает с ошибкой etckeeper
**Симптом:** В логе задачи `upgrade_mysql_php_*` ошибка:

```text
fatal: [testserver]: FAILED! => {"changed": true, "cmd": "/usr/bin/etckeeper commit ...", "stderr": "etckeeper warning: etckeeper is not yet enabled for /etc\netckeeper warning: run etckeeper init to enable it"}
```
**Причина:** etckeeper не был инициализирован.

**Решение:**

```bash
etckeeper init
```
После выполнения появится сообщение `[hooks]`. Повторить обновление MySQL.

### Ошибка при обновлении платформы

```text 
Interface "Bitrix\Booking\Interfaces\ProviderInterface" not found
```
**Симптом:** При обновлении через Marketplace процесс останавливается на ~25% с указанной ошибкой.

**Причина:** Модуль «Бронирование» (booking) пытается установиться раньше зависимых модулей (crm, main). Это баг последовательности обновлений в самом Битрикс24.

**Решение:**
Нажать **«Проверить обновления»**, затем повторно **«Установить рекомендуемые обновления»**. Со второго раза обновление проходит успешно.

Если ошибка повторяется:

1. Временно отключить модуль:

   ```bash
   mv /home/bitrix/www/bitrix/modules/booking /home/bitrix/www/bitrix/modules/booking.bak
   ```
2. Выполнить обновления

3. Вернуть модуль:

   ```bash
   mv /home/bitrix/www/bitrix/modules/booking.bak /home/bitrix/www/bitrix/modules/booking
   ```
4. Обновить модуль отдельно через `Marketplace → Установленные решения`
5. Так же можно установить модуль отдельно через `Настройки → Настройки продукта → Модули`

### Ошибка при выгрузке единиц измерения: м2
**Симптом:** В логе обмена:

```text
Произошла ошибка на сервере. Ответ сервера: failure. Ошибка импорта метаданных. Ошибка добавления новой единицы измерения (код единицы: м2).
```
**Причина:** В Битриксе уже существует единица измерения с кодом "м2" с другими параметрами.

**Решение (для чистовой интеграции):**

1. В админке Битрикса: `Магазин → Настройки → Единицы измерения`

2. Найти единицу с кодом "м2" и удалить её

3. В 1С: `Сервис → Идентификаторы объектов → Невыгруженные пакеты`

4. Удалить пакеты с единицами измерения

5. Повторить обмен

**Для тестового стенда:** Ошибку можно игнорировать — выгрузка происходит корректно.

### Ошибка обмена заказов

```text
Переменная не определена (ПакетнаяОбработкаТабличнойЧастиКлиентСервер)
```

**Симптом:** При попытке выгрузить заказы из Битрикс в 1С возникает ошибка инициализации модуля.

**Причина:** Учебная версия платформы 1С:Предприятие работает только в файловом режиме и не поддерживает клиент-серверную архитектуру, необходимую для полноценной обработки документов.

**Решение:** Для полноценной работы с заказами требуется:

 - Полноценная лицензионная версия 1С:Предприятие

 - Подписка ИТС для обновления конфигурации

 - Развёртывание 1С в клиент-серверном режиме

Для тестового стенда эта ошибка является ожидаемым ограничением учебной версии.

### После перезагрузки хоста виртуалка не получает IP-адрес

**Симптом:** Интерфейс `eth0` в состоянии `UP`, но IP-адрес отсутствует. Команда `dhclient` не получает ответа от DHCP-сервера.

**Причина:** Роутер мог перераспределить IP-адреса или «забыть» MAC-адрес виртуальной машины.

**Решение:**

1. Запросить IP вручную:

   ```bash
   dhclient -v eth0
   ```
2. Если не помогает — настроить статический IP (подставить свои адреса и названия интерфейсов):

   ```bash
   nmcli connection modify eth0 ipv4.method manual ipv4.addresses 192.168.1.78/24 ipv4.gateway 192.168.1.1 ipv4.dns 8.8.8.8

   nmcli connection down eth0 && nmcli connection up eth0
   ```

**Профилактика:**

 - Использовать тип адаптера virtio-net (вместо e1000)

 - Выключать виртуальную машину только через меню BitrixVM (2 → 5. Shutdown server)

### Общая рекомендация: корректное завершение работы ВМ
Для предотвращения проблем с сетью и MySQL всегда выключать виртуальную машину через меню BitrixVM:

`2. Configure localhost settings → 5. Shutdown server`

Только после этого закрывать окно VirtualBox или выключать хост-компьютер.
